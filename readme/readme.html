Visual C++ para Desenvolvimento em Linux

Configurando seu projeto para Desenvolvimento para Linux

Com essa carga de trabalho, você pode criar o código C++ para servidores, desktops e dispositivos Linux. Você pode gerenciar suas conexões com esses computadores de dentro do VS. O VS copiará suas fontes automaticamente e as compilará remotamente. Além disso, ele pode iniciar o aplicativo com o depurador. Nosso sistema de projeto dá suporte a arquiteturas de destino específicas, incluindo o ARM.

Conectando ao Linux

Pré-requisitos

Hoje, há suporte para o build remoto apenas em computadores de destino Linux. Não estamos limitados há distribuições específicas do Linux, mas dependemos da presença de algumas ferramentas. Especificamente, precisamos de openssh-server, g++, gdb e gdbserver. Use seu gerenciador de pacotes favorito para instalá-las. Por exemplo, em sistemas baseados em Debian: sudo apt-get install openssh-server g++ gdb gdbserver

Primeira conexão

Na primeira vez em que você direcionar a um computador Linux, serão solicitadas informações de conexão. Essa solicitação será disparada durante o build do projeto.

Adicionando e removendo conexões

Para adicionar uma nova conexão, acesse Ferramentas > Opções e procure Conexão. O Gerenciador de Conexões estará em Multiplataforma. Nesse local você pode adicionar e remover conexões.

Para alterar qual conexão um projeto está usando, acesse as configurações gerais das propriedades do projeto e atualize a opção Computador de Build Remoto.

Propriedades do Project

Todas as opções necessárias para controlar a compilação do C++ estão expostas nas páginas de propriedades do projeto. Abordaremos algumas informações específicas sobre como isso funciona no Linux. Primeiro, nas configurações gerais, você verá que a raiz remota está definida como ~/projects/ por padrão e que estamos definindo o diretório do projeto remoto para que ele corresponda ao nome do projeto nesse local.

Olhando as Configurações gerais do projeto, veja como os diretórios de saída e intermediários foram configurados. Além disso, você verá que esse projeto foi configurado como um aplicativo, portanto, o executável está em bin/x64/Debug/ como ConsoleApplication1.out. Observe que nos tipos de configuração também há suporte para bibliotecas estáticas e dinâmicas.

Adicione outras dependências de biblioteca na página de propriedades do Vinculador > Entrada.

Você pode passar comandos de pré-inicialização ao depurador para realizar tarefas, como iniciar aplicativos gráficos no computador remoto do Linux.

Você também pode enviar eventos de pós-build para controlar o comportamento remoto, como neste exemplo que exporta um PIN gpio para ser usado sem a necessidade de executar o executável como superusuário.
